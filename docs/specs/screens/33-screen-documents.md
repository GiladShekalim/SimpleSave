# Screen 33 — Documents Tab

## Purpose

Enable clients to upload, view, and track the status of all required and optional documents for their mortgage application. Authorization letters are managed here as a special sub-section. A progress gate prevents advancing to Principal Approval until all required documents are advisor-approved.

---

## Who Sees This / Access

- Authenticated clients with any tier (Tier 1, 2, or 3)
- **Locked** for clients with no tier — shows padlock overlay (see States section)
- **Locked** until lifecycle status ≥ TIER_SELECTED
- Advisors can view all documents, change statuses, add rejection reasons, and add new document requirements

---

## Layout Overview

Full-width content area. From top to bottom:
1. Authorization Letters section (always shown first, if applicable)
2. Progress bar / gate indicator
3. "Required Documents" section
4. "Additional Documents" section
5. "Next Step" button (bottom of page)

Document list uses a card-list layout — each document is a full-width row card.

---

## Sections / Components

### Authorization Letters Section (ייפויי כוח)

Shown at the top with a prominent **"Action Required / נדרשת פעולה"** amber badge, only when authorization letters have been generated and not all are signed.

Once all letters are signed and uploaded, this section collapses into a read-only "Completed" summary card.

**Per bank letter row:**

| Element | Details |
|---|---|
| Bank name (Hebrew) | e.g. "בנק לאומי" |
| Letter type label | "ייפוי כוח" |
| Status badge | לא נחתם (Not Signed, grey) / הועלה (Uploaded/Signed, green) |
| "הורד מכתב" / Download Letter | Button — downloads pre-generated PDF authorization letter for this bank |
| "העלה מכתב חתום" / Upload Signed Letter | Button — opens file picker. Only shown if status ≠ Uploaded |

**Auto-trigger:** When all authorization letters for all participating banks have been uploaded → system sets lifecycle status to AUTHORIZATION_SIGNED.

---

### Progress Bar / Gate Indicator

Full-width bar below authorization letters section.

Content:
- Text: "X מתוך Y מסמכים נדרשים אושרו" / "X of Y required documents approved"
- Progress bar: fill percentage = X / Y × 100
- Color: red (0–50%), amber (51–99%), green (100%)

**Gate completion state (all required docs approved):**
- Bar turns fully green with checkmark
- Success message: "כל המסמכים הנדרשים אושרו — ניתן להמשיך לאישור עקרוני." / "All required documents approved — proceed to Principal Approval."
- "Next Step" button at bottom of page becomes enabled and highlighted

---

### Document Row Structure

Each document (required or additional) is rendered as a row card.

| Element | Type | Details |
|---|---|---|
| Document name | Text (Hebrew) | Clickable link only when status = Approved — opens Document Viewer Modal |
| Status badge | Visual badge | see Status badge values below |
| ⓘ Info button | Icon button | On click: opens popover/tooltip with Hebrew explanation of the document and why it is needed |
| Upload area | Button + drag-drop zone | Visible only when status ≠ Approved. Client can click "העלה מסמך" / Upload Document or drag a file onto the row. |
| Rejection reason | Text (red) | Displayed below row only when status = Rejected. Shows advisor's rejection reason. |
| Re-upload button | Button | "העלה שוב" / Re-upload. Shown when status = Rejected OR status = Uploaded (to replace a previously uploaded file). |
| Upload spinner | Inline spinner | Shown during active file upload, replaces upload button temporarily |

**Status badge values:**

| Status Key | Hebrew Label | Color |
|---|---|---|
| REQUIRED | נדרש | Grey |
| UPLOADED | הועלה | Blue |
| APPROVED | אושר | Green |
| REJECTED | נדחה | Red |

---

### Document List — Required Documents (מסמכים נדרשים לאישור)

Section header: "מסמכים נדרשים לאישור" with badge count of remaining unapproved docs.

The list of required documents is **dynamic** — generated based on:
- Borrower employment status (employee vs. self-employed vs. controlling shareholder)
- Loan type
- Property registration type
- Whether a rental contract applies
- Manual additions by advisor

**Base required document list (shown to all clients):**

| Document Hebrew Name | Notes |
|---|---|
| ייפוי כוח — [bank name] | One letter per bank being applied to (generated by system) |
| דפי חשבון בנק — 3 חודשים | 3 months of bank statements (per checking account) |
| תלושי שכר — 3 חודשים | 3 months of pay slips — shown only IF employment_status = Employee |
| צילום תעודת זהות | ID copy (front + back pages) |
| פירוט הלוואות | Loan details document from bank |
| דוח קרן השתלמות | Savings fund statement — shown only IF savings_fund = כן |

**Additional documents (conditional by borrower profile):**

| Document | Condition |
|---|---|
| אישור הכנסות לעצמאי / Income confirmation | employment_status = Self-Employed OR Controlling Shareholder |
| שומת מס אחרונה / Tax assessment | employment_status = Self-Employed OR Controlling Shareholder |
| חוזה שכירות / Rental contract | Client currently renting (rent_payment > 0) |
| חוזה רכישה / Purchase contract | purchase_status = Signed Contract |
| פרטי ערב / Guarantor details | Advisor manually adds if guarantor is required |
| שמאות / Appraisal report | Advisor manually adds |
| נסח טאבו / Tabu extract | property_registration = Tabu |
| [Custom documents] | Advisor can add free-text custom document requirements |

### Document List — Additional Documents (מסמכים נוספים)

Section header: "מסמכים נוספים" — these documents are not required to gate Principal Approval but are useful for the application.

Same row structure as required documents. "Optional" label shown in small text on row.

---

### Upload Interaction Detail

1. Client clicks "העלה מסמך" or drags a file onto the row
2. File picker or drag event captures the file
3. **Client-side validation (immediate, before upload):**
   - File type: must be PDF, JPG, or PNG — otherwise inline error: "ניתן להעלות רק קבצי PDF, JPG, PNG."
   - File size: must be ≤ 10MB — otherwise inline error: "גודל הקובץ חורג מ-10MB."
4. If valid: row enters "Uploading" state — spinner on row, "מעלה..." label
5. On upload success: status badge changes to "Uploaded" (blue). "Upload" button replaced by "Re-upload" button.
6. On upload failure (network error): error toast "ההעלאה נכשלה. אנא נסה שוב." — upload button re-enabled.

---

### Document Viewer Modal

Triggered by clicking on an approved document's name link.

| Element | Details |
|---|---|
| Modal type | Full-screen overlay (100vw × 100vh), dark backdrop |
| PDF rendering | PDF.js inline rendering (page-by-page scroll) |
| Image rendering | Full-size image with scroll, centered |
| Page navigation | Prev / Next buttons for PDFs, page counter "עמוד X מתוך Y" |
| Close button | X button top-right |
| Download button | "הורד מסמך" — triggers browser download of the file |
| Zoom controls | + / − buttons or pinch gesture (mobile) |

The viewer is read-only. Clients can only view and download their own submitted documents.

## Buttons

| Button | Location | Action | Enabled Condition |
|---|---|---|---|
| הורד מכתב / Download Letter | Auth letters section, per bank | Downloads PDF | Always enabled when letter is generated |
| העלה מכתב חתום / Upload Signed Letter | Auth letters section, per bank | Opens file picker | Shown when letter not yet uploaded |
| העלה מסמך / Upload Document | Per document row | Opens file picker / drag zone | Shown when status ≠ Approved |
| העלה שוב / Re-upload | Per document row | Replaces existing upload | Shown when status = Rejected or Uploaded |
| View (document name link) | Per document row | Opens viewer modal | Enabled only when status = Approved |
| המשך / Next Step | Bottom of page | Navigates to Principal Approval tab | Enabled only when all required documents = Approved |

## States

### Locked (no tier)

- Full tab content replaced by padlock overlay
- Message: "שדרגו את התכנית שלכם כדי לגשת לעלאת מסמכים" / "Upgrade your plan to access document upload"
- "שדרג עכשיו" / Upgrade Now button → 46-screen-tier-selection-pricing.md

### Initial state (TIER_SELECTED, no documents uploaded yet)

- Authorization letters section visible (if letters have been generated) OR placeholder: "ייפויי הכוח יופיעו כאן לאחר שנכין אותם עבורך" / "Authorization letters will appear here after we prepare them."
- All required documents show "נדרש" (Required) grey badge
- Progress bar at 0%
- "Next Step" button disabled

### Partial progress

- Mix of Required / Uploaded / Approved / Rejected statuses
- Progress bar reflects approved / required ratio
- If any documents are Rejected: rejected document rows have red styling, rejection reason text visible

### All required documents approved

- Progress bar 100% green
- "כל המסמכים הנדרשים אושרו!" success banner
- "Next Step" button enabled
- Lifecycle status advances to DOCUMENTS_APPROVED (server-side trigger)

### Authorization letters pending signature

- Auth letters section shown with amber "Action Required" badge
- Reminder banner at top: "נדרשת פעולה: חתמו והעלו את ייפויי הכוח לפני שנגיש לבנקים"
